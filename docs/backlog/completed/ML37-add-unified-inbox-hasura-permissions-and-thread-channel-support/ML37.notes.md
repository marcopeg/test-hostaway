# Execution Notes - Add unified inbox Hasura permissions and thread channel support
**Task**: ./ML37.task.md
**Plan**: ./ML37.plan.md

## Decisions

- User-role reads are scoped by `x-hasura-tenant-id`, not `x-hasura-user-id`.
- `x-hasura-user-id` is used for pending outgoing message inserts by presetting `operator_id`.
- `data.threads.channel_id` is mandatory for all threads.
- Last-message data stays on the messages relationship using ordering and `limit: 1`.
- `data.channels` is treated as shared reference data and is readable by the `user` role.
- Because `auth.users` has no tenant column, operator rows are readable to `user` only when connected to messages or pending messages in the current tenant.
