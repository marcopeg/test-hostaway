# Execution Notes - Build unified inbox split-view UI and navigation shell
**Task**: ./YW27.task.md
**Plan**: ./YW27.plan.md

## Decisions

- Accepted plan and execution were inferred from the operator request: `plan and execute`.
- The webapp GraphQL headers need a concrete tenant for the unified inbox user-role filters, so the implementation uses seeded tenant `gre` and seeded operator `amy` for local development defaults.
- `total unanswered` and unread-style thread emphasis are calculated from the merged latest logical message across persisted `messages` and outgoing `messages_pending`.

## Problems Encountered

- The existing top user query requested `auth_users.username`, but the current schema exposes `name` and `icon`; this had to be corrected for the app shell to load real user data.
- The browser plugin tool was not exposed in this session, and the `node_repl` Playwright fallback reported `Module not found: playwright`, so visual browser screenshot verification could not be completed.

## Deviations From Plan

- Browser verification was limited to Vite route HTTP responses and build/lint checks because no browser automation runtime was available.
