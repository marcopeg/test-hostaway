## Questions

### 1. What should the root route `/` do after removing the todo demo?

The task can either redirect `/` to `/inbox`, or keep `/` as a registered Work in progress page such as Overview. Redirecting makes the app open directly into the implemented inbox; keeping `/` as Overview preserves the menu structure more literally.

Answer:

redirect to /inbox

### 2. Are these main menu paths acceptable?

Proposed route paths: `/overview`, `/calendar`, `/reservations`, `/smart-locks`, `/inbox`, `/tasks`, `/reviews`, `/guest-payments`, `/financial-reporting`, `/owner-statements`, `/listings`, `/channel-manager`, and `/settings`. Every route except `/inbox` would render the shared Work in progress page.

Answer:

yes

### 3. What is the MVP definition of `total unanswered` and unread-style thread emphasis?

Recommended MVP: a thread is unanswered when its latest persisted message is from a guest, and the thread row gets bold unread-style emphasis for the same condition. This does not require a new unread/read state yet.

Answer:

yes, but make sure you also pull related messages from the "messages_pending" so that you can then merge logically with an outgoing message.

### 4. Should the selected thread route be `/inbox/$threadId`?

Using `/inbox/$threadId` keeps the selected conversation in the URL and lets `/inbox` remain the stats/dashboard state. If you prefer a different shape, specify it here.

Answer:

good
