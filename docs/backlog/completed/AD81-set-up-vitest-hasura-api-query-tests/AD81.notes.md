## Decisions

- Use the webapp package as the Vitest host, matching `TESTING.md`.
- Keep the first test query-only and limited to persisted `messages`.
- Reset backend state with the existing Makefile workflow from the test process.
- Force `TERM=xterm` for reset child processes so the public Make targets that call `clear` work under Vitest.

- `make migrate.rebuild` initially failed from the Vitest child process because `clear` did not have a usable terminal setting.
- A sandboxed validation run could not use Hasura CLI localhost requests; the suite passed once run with approved local backend access.
