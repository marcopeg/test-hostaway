# Create Hostaway UI moodboard signature

## Elevator's Pitch

Analyze the provided Hostaway app screenshots and distill them into a reusable local Codex skill at `.agents/skills/hostaway-ui`, plus Tailwind theme tokens in `apps/webapp/src/index.css`, so future frontend work can produce a recognizable Hostaway-like operational interface quickly.

## Business Gain

A shared Hostaway UI skill gives future frontend work concrete visual, spacing, typography, and component guidance before screens are implemented. This reduces subjective design drift and helps the app feel coherent with the Hostaway-inspired unified inbox and messaging domain while preserving fast prototyping speed.

## Current State

The backlog includes a Hostaway unified inbox data-model task, and this task folder now includes five screenshots:

- `messaging.png`: canonical source for the app UI signature.
- `reservations.png`: supporting source for the admin shell, dashboard cards, sidebar, and list density.
- `calendar.png`: supporting source for dense operational tables, filters, tabs, and channel/status treatments.
- `card.png`: supporting source for compact reservation-card structure.
- `home.png`: marketing screenshot; do not use as a canonical app UI source.

The frontend currently uses Tailwind v4 through `apps/webapp/src/index.css`, which only imports Tailwind.

## Desired State

Create `.agents/skills/hostaway-ui/SKILL.md` with a description that makes agents select it for Hostaway-like app UI work, especially unified inbox and messaging workflows. The skill should extract a Hostaway-like feeling from the screenshots without copying proprietary assets or exact UI details.

The skill should guide agents using Tailwind toward:

- Messaging-first layouts based mainly on `messaging.png`.
- Supporting admin-shell, dashboard, calendar, and reservation-card patterns from `reservations.png`, `calendar.png`, and `card.png`.
- Color scheme, approximate font choices, spacing rhythm, title/subtitle hierarchy, dense list/table treatment, sidebars, panels, badges, buttons, AI reply surfaces, and operational state styling.
- Practical Tailwind usage and examples that are close to Hostaway where useful but do not block fast prototyping.

Update `apps/webapp/src/index.css` with Tailwind v4 theme tokens and any base-layer CSS needed to make the Hostaway-inspired palette, typography, radius, border, surface, and status styles available to future frontend implementation.

## Definition of Success

- `.agents/skills/hostaway-ui/SKILL.md` exists and can be selected by agents for Hostaway-like frontend work.
- The skill description explicitly names the contexts that should trigger it, including Hostaway-like operational UI, unified inbox, guest messaging, reservations, calendar, and admin-shell work.
- `apps/webapp/src/index.css` defines Tailwind v4 tokens or base styles for the extracted Hostaway-like UI signature.
- The guidance prioritizes `messaging.png` and uses `reservations.png`, `calendar.png`, and `card.png` as supporting evidence.
- The guidance excludes the marketing homepage as a canonical source for app UI.
- The output is usable for fast React/Tailwind prototyping without requiring pixel-perfect copying.
