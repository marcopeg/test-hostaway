# Execution Notes - Set up Playwright UI test suite
**Task**: ./AW48.task.md
**Plan**: ./AW48.plan.md

## Decisions

- Use Playwright webServer to start Vite on port 5173 and reuse an existing local frontend server outside CI.
- Run the Playwright project with one worker because the suite shares a single local frontend/backend stack.
- Assert the overview work-in-progress state through visible text.
- Add narrow data attributes to rendered inbox threads and message bubbles so tests can verify dynamic seeded data without depending on guest names.
- Ignore local Playwright output directories in git.

## Problems Encountered

- The inbox UI rendered zero threads during Playwright validation because Hasura rejected the existing inbox subscription with `Data limited subscriptions must select one top level field`.

## Deviations From Plan

- Switched the inbox read operation from a subscription to a normal query, with an explicit network fetch on route mount, so the UI can render seeded thread data through the browser test path and after inbox thread navigation.

## Additional Requests

n/a

## Known Limitations

- `make test.ui` assumes the backend stack has already been started and initialized with seed data.
- The local Playwright Chromium browser cache must exist; it can be installed with `npx playwright install chromium`.
