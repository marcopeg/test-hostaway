# Refinement Questions v1 - Set up Playwright UI test suite

Answer inline under each question, then return to chat and say `next round`.

## Questions

### 1. Should `make test.ui` rely on Playwright's `webServer` to start the Vite frontend, or should it require `make app.start` to already be running?

Answer:
Use Playwright's `webServer` to start the Vite frontend and reuse an existing local server outside CI.

### 2. For the overview test, should the assertion validate exact visible copy such as "Work in progress", or should implementation add a stable `data-testid` for the overview work-in-progress state?

Answer:
Validate the visible "Work in progress" copy for the overview test. Add stable test ids only where visible text is duplicated or not enough to prove the requested behavior.

### 3. For the inbox test, should it open a specific seeded thread when available, or is it enough to open the first visible thread and assert its first message is from a guest?

Answer:
Open the first visible thread and assert its first rendered message is from a guest.
